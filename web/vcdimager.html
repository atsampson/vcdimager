Introduction to GNU VCDImager

GNU VCDImager is a freely available tool for generating Video-CD BIN/CUE images from properly encoded MPEG streams, which can be played on standalone VCD players, on DVD players and on most computer systems.

Downloading GNU VCDImager

GNU VCDImager can be found on in the subdirectory /gnu/vcdimager/ on your favorite GNU mirror. For other ways to obtain GNU VCDImager, please read How to get GNU Software

Documentation

GNU VCDImager documentation can be found at http://www.gnu.org/manual/manual.html

Mailing Lists/Newsgroups

GNU VCDImager now has two mailing lists: <help-vcdimager@gnu.org> and <bug-vcdimager@gnu.org>.

The main discussion list is <help-vcdimager@gnu.org>, and is used to discuss all aspects of GNU VCDImager.

Announcements about GNU VCDImager are made on <info-vcdimager@gnu.org>.

There is also a special list used for reporting bugs, <bug-vcdimager@gnu.org>. For details on submitting a bug report, please see the section Report a Bug below.

To subscribe to any GNU VCDImager mailing list, please send an empty mail with a Subject: header line of just "subscribe" to the relevant -request list. For example, to subscribe yourself to the main list, you would send mail to <help-vcdimager-request@gnu.org> with no body and a Subject: header line of just "subscribe".

Request an Enhancement

If you would like any new feature to be included in future versions of GNU VCDImager, please send a request to <help-vcdimager@gnu.org>

Please remember that development of GNU VCDImager is a volunteer effort, and you can also contribute to its development. For information about contributing to the GNU Project, please read How to help GNU.

Report a Bug

If you think you have found a bug in GNU VCDImager, then you should send as complete a report as possible to <bug-vcdimager@gnu.org>. Ideally, you should include the text you get by running config.guess, the text you see when you run , and configure, and if you can, a patch made with diff -u5 which fixes the problem.

Maintainer

GNU VCDImager is currently being maintained by Herbert Valerio Riedel <hvr@gnu.org>.
